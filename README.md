# Система формирования коммерческих предложений на крепёж

Автоматизированная система формирования коммерческих предложений (КП) с использованием **локальной языковой модели (LLM)**.

| Параметр | Значение |
|----------|----------|
| **Заказчик** | ООО «Веб Успех» |
| **Исполнитель** | ИП Стасиньски Павел Кшиштоф |
| **Договор** | № 12.05.2026 от 12.05.2026 г., г. Москва |
| **Срок разработки** | 3 календарные недели (с момента предоплаты и передачи всех материалов) |
| **Развёртывание** | Облако Selectel (или иной отечественный провайдер) |

---

## Назначение

Система предназначена для операторов отдела продаж: быстрая обработка входящих заявок на крепёж, автоматический подбор позиций из базы наличия и аналогов, ручная корректировка и выпуск КП в корпоративном формате Заказчика.

**Цели:**

- сократить время обработки одной заявки с часов до минут;
- снизить ошибки ручного подбора, расчёта сумм и проверки наличия;
- обеспечить конфиденциальность данных за счёт обработки на защищённой инфраструктуре без передачи в сторонние облачные LLM-сервисы;
- упростить работу оператора через чат-интерфейс и табличный редактор КП.

---

## Функциональные возможности

| № | Функция | Описание |
|---|---------|----------|
| 2.1 | **Ввод заявки** | Текст в чате; drag&drop файлов: `.txt`, `.eml`, `.pdf`, `.xlsx`, `.png`, `.jpg` |
| 2.2 | **Парсинг запроса** | Локальная LLM извлекает наименования, характеристики (размер, резьба, покрытие, ГОСТ/DIN), количество и особые требования |
| 2.3 | **Сверка с базой** | Поиск точных совпадений и технических аналогов по ГОСТ/DIN и характеристикам при отсутствии точного совпадения |
| 2.4 | **Черновик КП** | Таблица: №, Наименование, Кол-во, Ед. изм., Цена с НДС, Сумма, Вес, Статус; автоподсчёт итогов по сумме и весу |
| 2.5 | **Ручная коррекция** | Изменение количества и цены; удаление/добавление позиций через поиск по базе; статусы; комментарии к позициям и КП |
| 2.6 | **Генерация КП** | Экспорт PDF по корпоративному шаблону: шапка, таблица, итоги, примечание об изменении цен, подпись, дата |
| 2.7 | **Логирование** | История заявок, черновиков и итоговых КП; повторное открытие; учёт правок оператора для последующего дообучения |

### Дополнительные правила (ТЗ)

- **OCR для PDF:** при сканированном PDF — автоматическое распознавание текста перед парсингом LLM; при низком качестве скана — уведомление оператора о ручной проверке.
- **Единицы измерения:** если в заявке указаны не штуки (например, кг), единица сохраняется без автоконвертации; статус позиции — **«Требует проверки»**; пересчёт выполняет оператор.

## Архитектура репозитория

Проект построен на базе монорепозитория с разделением ответственности:

| Каталог | Назначение |
|---------|------------|
| `frontend/` | Веб-интерфейс (Vite + React): чат, загрузка файлов, табличный редактор КП |
| `server/` | API (Node.js + Express): LLM, БД, логика сверки, экспорт PDF, история заявок |
| `collector/` | Обработка и парсинг входящих документов (в т.ч. OCR для PDF) |
| `docker/` | Сборка и инструкции для развёртывания на Selectel |

Данные Заказчика обрабатываются на инфраструктуре под управлением Исполнителя; передача содержимого заявок в сторонние публичные LLM API **не предусмотрена** договором.

---

## Данные, которые предоставляет Заказчик

До начала разработки необходимо передать:

1. **База крепежа** (Excel, CSV или согласованный формат) с полями: наименование, характеристики, наличие, цена с НДС, вес, стандарт.
2. **Корпоративный шаблон КП** (Word / Excel / PDF).
3. **Доступы** к облачной инфраструктуре (после регистрации у провайдера, указанного Исполнителем).
4. **Примеры заявок** — не менее 5 реальных входящих запросов для настройки и тестирования парсера.

Срок исполнения **не исчисляется**, пока не поступила предоплата и не переданы все перечисленные материалы.

## Развёртывание (продакшен)

Целевая среда — **Selectel** (или другой отечественный провайдер по решению Исполнителя). Доступ Заказчику — через веб-интерфейс.

Подробности сборки и запуска в контейнере:

- [`docker/HOW_TO_USE_DOCKER.md`](./docker/HOW_TO_USE_DOCKER.md)

### Railway

Обязательные переменные и том (см. `railway.toml`):

| Переменная | Значение |
|------------|----------|
| `STORAGE_DIR` | `/data/storage` |
| `DATABASE_URL` | `file:/data/storage/offerkp.db` |
| `JWT_SECRET` | случайная строка ≥12 символов (не менять между деплоями) |

Том: mount path **`/data`**, внутри приложение использует **`/data/storage`**. Без `DATABASE_URL` на томе SQLite Prisma пишет в эфемерный `/app/server/storage` — пользователи и админ сбрасываются при каждом деплое.

Минимальные требования к оборудованию согласуются Сторонами отдельно после подписания договора.

## Локальная разработка

Требования: **Node.js ≥ 18**, **Yarn**.

```bash
# Установка зависимостей и копирование .env из примеров
yarn setup

# Заполнить server/.env.development, frontend/.env, collector/.env

# Запуск в отдельных терминалах:
yarn dev:server
yarn dev:frontend
yarn dev:collector

# Или одной командой:
yarn dev:all
```

Полезно: [`server/storage/documents/DOCUMENTS.md`](./server/storage/documents/DOCUMENTS.md) — работа с документами в хранилище.

### База данных (Prisma)

```bash
yarn prisma:setup    # generate + migrate + seed
yarn prisma:reset    # сброс SQLite (dev)
```

### Телеметрия

В продакшене для Заказчика рекомендуется отключить внешнюю телеметрию:

```env
DISABLE_TELEMETRY=true
```

Либо в интерфейсе: **Privacy** → отключить телеметрию.
